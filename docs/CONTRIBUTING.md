# Contributing Guide

Thanks for your interest in SchemaForge.

## Development Workflow

1. Fork the repository.
2. Create a feature branch.
3. Make small, reviewable changes.
4. Add or update tests.
5. Run the full test suite before opening a pull request.

## Local Commands

```bash
dotnet build SchemaForge.slnx
dotnet test SchemaForge.slnx
```

## Contribution Standards

- Keep commits focused
- Prefer clear endpoint contracts
- Add documentation for user-facing behavior
- Include tests for validation logic

## Early Contribution Areas

- JSON Schema validation workflows
- Report formatting
- Export features
- Documentation improvements
- Test coverage expansion
